## GPIO 概述
**GPIO** = General Purpose Input Output，即**通用输入输出端口**，俗称 IO 口。
### 基本参数

| 参数 | 值 |
|------|-----|
| 引脚电压范围 | 0V ~ 3.3V |
| 逻辑低电平 | 0V ~ 2V（即数据 0） |
| 逻辑高电平 | 2V ~ 3.3V（即数据 1） |
| 部分引脚**5V 容忍**（FT） | 可输入 5V 电压仍被认作高电平，但输出最大只能 3.3V |

> 带 **FT**（Five Volt Tolerant）标识的引脚可容忍 5V 输入，不带 FT 的引脚只能接 3.3V。

### GPIO 的应用场景
**输出模式用途：**
- 控制 LED 亮灭
- 驱动蜂鸣器
- 模拟通信协议时序（I2C、SPI、自定义协议）
- 驱动功率设备（需外接驱动电路）

**输入模式用途：**
- 读取按键状态
- 读取模块电平信号（光敏/热敏电阻模块等数字输出）
- 配合 ADC 采集模拟电压
- 模拟通信协议接收数据

## GPIO 基本结构（STM32 内部）
### 整体架构图
![GPIO基本结构图|500](GPIO基本结构图.png)
> 图：GPIO基本结构图
[[1、STM32简介#STM32 系统结构|STM32 所有 GPIO 均挂载在 **APB2 外设总线**上]]，每个 GPIO 外设有 16 个引脚（Pin 0 ~ Pin 15），命名如 PA0、PA1 ... PA15、PB0 ... PB15。

> [!info]+ GPIO 端口与引脚：为什么是 16 个？PA1 是哪个引脚？
>
> **GPIO 端口（Port）的 16 个引脚是芯片内部的设计**，每个 GPIO 外设（如 GPIOA）在硬件上确实有 16 根内部信号线，分别叫 Pin 0、Pin 1 ... Pin 15，对应寄存器中的 16 个位。这是 ARM Cortex-M 内核架构规定的标准设计，每个 GPIO 端口固定为 16 位。
>
> **PA1 = Port A 的第 1 号引脚**，这是芯片内部 GPIO 外设的信号名称，与芯片物理封装引脚号是两个不同的概念：
>
> - **PA1**（Port A Pin 1）：GPIO 外设内部的信号名称，代表 GPIOA 模块的第 1 号引脚
> - **引脚编号（如第 23 脚）**：芯片物理封装上的引脚编号
>
> 两者通过芯片内部的**引脚复用**机制对应，一个物理引脚可以复用为多种功能，具体功能由配置决定。
>
> **以 STM32F103C8T6（LQFP-48 封装，俗称"蓝 pill"）为例**，共有 48 个物理引脚：
>
> | 物理引脚（部分） | 复用功能 |
> |----------------|---------|
> | 第 10 脚 | PA0 |
> | 第 11 脚 | PA1 |
> | 第 12 脚 | PA2 |
> | ... | ... |
> | 第 15 脚 | PA5（LED） |
> | 第 16 脚 | PA6 |
> | 第 29 脚 | PB11 |
> | ... | ... |
>
> 该芯片只引出了 GPIOA（PA0~PA15）、GPIOB（PB0~PB15）、GPIOC（PC13~PC15）中的部分引脚，并非所有 16×3=48 个引脚都引出。GPIOC 甚至只引出了 PC13~PC15 三个引脚。
>
> **为什么设计 16 个而不是按需设计？** ARM 架构要求标准化、同一个 GPIO 外设模块可以在不同芯片型号中通用，设计为 16 位也留有扩展余地。尽管某些型号只引出了其中一部分，但内部寄存器结构、驱动程序均保持一致，降低了开发迁移成本。
>
> **命名对照**：`PA1` 中的 `P` 是 Port，`A` 是端口号，`1` 是引脚序号，全称是 **Port A Pin 1**。PB1 同理为 Port B Pin 1。
>
> **为什么只引出部分 GPIO 引脚？** 这是芯片物理封装与内部外设数量博弈的结果。以 STM32F103C8T6 为例，芯片只有 48 个物理引脚，而 STM32 内部集成了非常多的外设（GPIOA/B/C、USART、SPI、I2C、ADC、TIM 等），每个外设都需要引脚。这些外设所需的引脚总数远超过 48 个物理引脚，因此芯片采用**引脚复用**（Pin Multiplexing）机制——同一个物理引脚可以复用到多个外设上。但即便如此，48 个引脚仍不够用，所以只能**优先引出最常用的 GPIO 信号**，其他内部信号（如 GPIOA 的 PA6~PA15 中的部分）就不从物理引脚引出。
>
> **没有被引出的 GPIO 内部引脚还能使用吗？** **不能**。芯片内部 GPIO 模块的信号线（如 PA12）如果没有从物理引脚引出，就没有任何方式可以将信号接入或输出。软件上可以配置对应寄存器的对应位，但物理上这些信号根本没有连接到外部，无法产生任何实际的电平变化。这类未引出的引脚只是芯片内部保留的内部信号，功能上等同于不存在。
>
> **部分引出（如 GPIOC 仅 PC13~PC15）的原因**：这是芯片厂家在设计 LQFP-48 封装时综合成本、引脚数量后的取舍。LQFP-48 封装引脚数量有限，必须将有限的引脚分配给最常用的功能，因此只引出了 GPIOA 和 GPIOB 的全部，以及 GPIOC 的 3 个（PC13 为备份寄存器时钟输出引脚，PC14/PC15 为低频晶振引脚）。
```
APB2 总线
   │
   └── GPIO 外设组（A / B / C / D / E / F / G）
            │
            ├── 16 个引脚（Pinx）
            │
            ├── 端口配置低寄存器（GPIOx->CRL）→ 控制 Pin 0~7
            ├── 端口配置高寄存器（GPIOx->CRH）→ 控制 Pin 8~15
            ├── 端口输入数据寄存器（GPIOx->IDR）→ 读取引脚电平
            ├── 端口输出数据寄存器（GPIOx->ODR）→ 控制输出电平
            ├── 端口位设置/清除寄存器（GPIOx->BSRR）→ 单独操作某位
            └── 端口配置锁定寄存器（GPIOx->LCKR）
```

### 寄存器说明

| 寄存器           | 作用                                         |
| ------------- | ------------------------------------------ |
| `GPIOx->CRL`  | 端口配置低寄存器，每 4 位配置一个引脚（Pin 0~7），共 32 位       |
| `GPIOx->CRH`  | 端口配置高寄存器，每 4 位配置一个引脚（Pin 8~15），共 32 位      |
| `GPIOx->IDR`  | 端口输入数据寄存器，只读，低 16 位有效                      |
| `GPIOx->ODR`  | 端口输出数据寄存器，读写，低 16 位有效                      |
| `GPIOx->BSRR` | 端口位设置/清除寄存器，高 16 位写 1 = 清零，低 16 位写 1 = 置 1 |
| `GPIOx->LCKR` | 端口配置锁定寄存器，可锁定配置防止意外修改                      |

> STM32 为 32 位单片机，内部寄存器均为 32 位，但 GPIO 端口仅 16 位，高 16 位未使用。

## GPIO 内部电路详解

![GPIO位结构图|500](GPIO位结构图.png)
> 图：GPIO位结构图
### 输入部分电路
**保护二极管：**
- 输入电压 > 3.3V → 上方二极管导通，电流流向 VDD，保护内部电路
- 输入电压 < 0V → 下方二极管导通，电流从 VSS 流出，保护内部电路
- 输入电压在 0~3.3V 之间 → 两个二极管均不导通，无影响

**上拉/下拉电阻（弱电阻）：**
- 上拉输入：默认高电平输入（引脚悬空时为高）
- 下拉输入：默认低电平输入（引脚悬空时为低）
- 浮空输入：上下拉均断开，引脚电平完全由外部决定，易受干扰

> 阻值较大，属于"弱上拉/弱下拉"，目的是不影响正常输入操作。

**施密特触发器：**
- 对输入波形进行整形，消除抖动
- 两个阈值：上限和下限
  - 输入 > 上限 → 输出瞬间变为高电平
  - 输入 < 下限 → 输出瞬间变为低电平
- 中间区域的波动不会引起输出跳变，有效避免因信号波动造成的误判

> 图中的"肖特基触发器"为翻译错误，原文为 **Schmitt Trigger（施密特触发器）**。

### 输出部分电路
**位设置/清除寄存器（BSRR）的用法：**
- 对某位**置 1**（不影响其他位）→ 在 `BSRR` 低 16 位对应位写 1
- 对某位**清零**（不影响其他位）→ 在 `BSRR` 高 16 位对应位写 1
> 注意：`BSRR` 高 16 位用于清零，低 16 位用于置 1。与直接写 `ODR`（会覆盖其他位）不同，用 `BSRR` 操作可单独控制某一引脚而不影响其他引脚。
## GPIO 八种工作模式

| 模式         | 类型   | 特点                            |
| ---------- | ---- | ----------------------------- |
| **浮空输入**   | 数字输入 | 上下拉均断开，引脚悬空，易受干扰              |
| **上拉输入**   | 数字输入 | 内部接上拉电阻，默认高电平                 |
| **下拉输入**   | 数字输入 | 内部接下拉电阻，默认低电平                 |
| **模拟输入**   | 模拟输入 | GPIO 电路全部关闭，引脚直连 ADC，无施密特触发器  |
| **开漏输出**   | 数字输出 | 仅 N-MOS 工作，高电平 = 高阻态（无驱动能力）   |
| **推挽输出**   | 数字输出 | P-MOS + N-MOS 均工作，高低电平均有强驱动能力 |
| **复用开漏输出** | 数字输出 | 同开漏，但由片上外设控制引脚（如 I2C 引脚）      |
| **复用推挽输出** | 数字输出 | 同推挽，但由片上外设控制引脚（如 USART TX）    |

> **重要规则**：除了模拟输入会关闭数字输入功能外，GPIO 的**所有输入电路始终有效**，即在任意输出模式下均可同时读取引脚电平。

| 对比项 | 推挽输出 | 开漏输出 |
|--------|---------|---------|
| 高电平驱动 | 强（P-MOS 导通，直接拉高到 VDD） | 无（P-MOS 无效，等效断开） |
| 低电平驱动 | 强（N-MOS 导通，直接拉低到 GND） | 强（N-MOS 导通，拉低到 GND） |
| 高电平实际电平 | 3.3V | **高阻态**（取决于外部上拉） |
| 常见用途 | 驱动 LED、常规数字输出 | I2C 通信、输出 5V 兼容电平 |

**开漏输出的特殊用途：**
- **I2C 通信**：多设备共用总线，开漏输出可避免总线冲突（任一设备拉低均有效，任一设备释放则由上拉决定高电平）
- **输出 5V 电平**：外部上拉电阻接到 5V 电源，则开漏输出高电平时引脚被外部电阻拉至 5V，可兼容 5V 设备

## GPIO 输出速度

| 速度配置 | 说明 |
|---------|------|
| 10 MHz | 低速，功耗低，适用于低速外设 |
| 2 MHz | 中速 |
| 50 MHz | **高速（常用）**，适用于高速通信如 SPI |

> GPIO 输出速度限制引脚最大翻转频率，主要出于降低功耗和增强稳定性的设计考量。一般对速度要求不高的场景直接配置 50MHz 即可。

## 外部设备电路

### LED（发光二极管）

**特性：** 正向通电点亮，反向不通电不亮。

**识别正负极：**
- 直插 LED：长脚 = 正极，短脚 = 负极
- 内部结构：较小的一半 = 正极，较大的一半 = 负极

**驱动电路：**

**低电平驱动（推荐）：**
```
VCC(3.3V) ──[限流电阻]──→ LED(+) ──→ LED(-) ──→ PA0
```
- PA0 输出低电平 → 电流从 VCC 流入 PA0，LED 亮
- PA0 输出高电平 → LED 两端无电势差，不亮

**高电平驱动：**
```
PA0 ──[限流电阻]──→ LED(+) ──→ LED(-) ──→ GND
```
- PA0 输出高电平 → 电流流入 GND，LED 亮
- PA0 输出低电平 → LED 不亮

> **限流电阻必须加**，一方面防止 LED 因电流过大烧毁，另一方面可调整 LED 亮度（电阻越大，亮度越暗）。

### 蜂鸣器

蜂鸣器分为**有源蜂鸣器**和**无源蜂鸣器**：

| 类型 | 内部 | 驱动方式 |
|------|------|---------|
| **有源蜂鸣器** | 自带震荡源，接 DC 即可持续发声 | 低电平触发响，高电平停止 |
| **无源蜂鸣器** | 无震荡源，需控制器提供脉冲 | 提供不同频率脉冲可发出不同音调 |

> 本教程使用有源蜂鸣器，接低电平发声，接高电平关闭。

**三极管驱动电路（蜂鸣器）：**

由于 STM32 GPIO 引脚直接驱动功率有限（大功率负载会加重 STM32 负担），通常使用三极管开关作为驱动电路：

| 电路类型 | 驱动逻辑 | 负载接法 |
|---------|---------|---------|
| PNP 三极管 | 基极给低电平导通 | 负载接在集电极与 VCC 之间 |
| NPN 三极管 | 基极给高电平导通 | 负载接在集电极与 GND 之间 |

> 负载应接在**电流流入端**：PNP 负载接上边（集电极），NPN 负载接下边（集电极），否则三极管可能因 BE 极压降不足而无法正常开启。

## 寄存器速查表

### CRL / CRH 配置（每引脚 4 位：CNF[1:0] + MODE[1:0]）

| CNF[1:0]（MODE=00 输入模式） | MODE[1:0]=00 | 说明 |
|-----------------------------|--------------|------|
| `00` | 模拟输入 |
| `01` | 浮空输入（默认） |
| `10` | 复用功能输入 |
| `11` | 下拉输入 |

| CNF[1:0]（MODE≠00 输出模式） | MODE=01/10/11 | 说明 |
|-----------------------------|---------------|------|
| `00` | **通用推挽输出** |
| `01` | 通用开漏输出 |
| `10` | 复用功能推挽输出 |
| `11` | 复用功能开漏输出 |

### 速度 MODE[1:0]

| 值    | 频率     |
| ---- | ------ |
| `01` | 10 MHz |
| `10` | 2 MHz  |
| `11` | 50 MHz |
