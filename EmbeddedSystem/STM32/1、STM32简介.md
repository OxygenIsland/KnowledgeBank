## STM32 概述

STM32 是 ST 公司基于 ARM Cortex-M 内核开发的 32 位微控制器。
- **ST**：ST 公司（意法半导体）
- **M**：Microcontroller，微控制器，即 MCU（单片机）
- **32**：32 位单片机，相比 8 位的 51 单片机性能更强

> ST 公司也推出了 8 位单片机 STM8，可作了解。

ARM Cortex-M 内核是 STM32 内部的核心部分，负责程序指令执行和运算，相当于整个芯片的 CPU。ST 公司从 ARM 公司获得内核授权后，封装整个芯片。其他半导体厂商也可以使用 ARM 内核生产自己的芯片。

STM32 常应用于嵌入式领域：
- **智能车**：读取光电传感器或摄像头数据，驱动电机前进和转弯
- **无人机**：读取陀螺仪数据，通过控制算法调节电机速度，保证飞行稳定
- **飞行机器人**：驱动舵机控制机器人关节运动
- **无线通信**：连接 2.4G 无线模块、蓝牙模块、WiFi 模块，实现无线通信
- **物联网**：通过无线模块通信，配合继电器控制 220V 电路通断
- **工业控制**：PLC 等工业设备中常见 STM32 的身影
- **娱乐电子产品**：如爱心流水灯等

STM32 功能强大、性能优异、片上资源丰富、功耗低，是经典的嵌入式微控制器。

## ARM 简介
ARM 既指 ARM 公司，也指 ARM 处理器内核。

**ARM 公司**是全球领先的半导体知识产权（IP）提供商——只设计内核，不生产实物，实际的内核由各大半导体厂商连同芯片一起制作出来。ARM 授权设计收取授权费作为盈利方式，这是一种开放合作、利益共享、风险共担的商业模式。目前全球超过 95% 的智能手机和平板电脑都采用 ARM 架构。
ARM 公司设计 ARM 内核，半导体厂商完善内核周边电路并生产芯片：

### ARM 内核分类
ARM 处理器分为 **A / R / M** 三个系列：

| 系列 | 全称 | 应用场景 |
|------|------|---------|
| **Cortex-A** | Application | 高端应用型领域，主要用于手机（苹果、高通、联发科芯片均采用 ARM A 系列），性能最高，发展最快 |
| **Cortex-R** | Real Time | 实时性很高的场景，如硬盘控制器等，应用场景较小 |
| **Cortex-M** | Micro Control | 单片机领域，如 STM32，性能相对较低，发展较慢 |

STM32 使用的就是 Cortex-M 内核，具体型号包括 M0、M1、M3、M4 等，不同型号内核性能不同：
- STM32 F0 → Cortex-M0
- STM32 F1 → Cortex-M3
- STM32 F3/F4 → Cortex-M4
- STM32 F7/H7 → Cortex-M7

## STM32 家族四大系列
STM32 全称为 32-bit ARM Cortex-M  microcontroller，目前主要有四个系列：

### 1. 高性能系列
- **STM32F2**：398 CoreMark，120MHz，Cortex-M3 内核
- **STM32F4**：608 CoreMark，180MHz，Cortex-M4 内核
- **STM32F7**：2020 CoreMark，216MHz，Cortex-M7 内核
- **STM32H7**：3224 CoreMark，550MHz Cortex-M7 + 240MHz Cortex-M4 双核微控制器（目前最强 STM32）

### 2. 主流系列（本次教程使用）
- **STM32F1**：177 CoreMark，72MHz，Cortex-M3 内核

### 3. 超低功耗系列
- STM32L0、STM32L1、STM32L4、STM32L5 等

### 4. 无线系列
- STM32WL、STM32WB 等

> 根据性能需求、功耗要求或无线通信需求，可选择不同系列。同一家族换型比换用其他芯片更轻松。

## STM32F103C8T6 详解
本教程使用的芯片型号：**STM32F103C8T6**

| 参数 | 值 |
|------|-----|
| 系列 | 主流系列 STM32F1 |
| 内核 | ARM Cortex-M3 |
| 主频 | 72MHz |
| RAM | 20KB（SRAM） |
| ROM | 64KB（Flash 闪存） |
| 供电电压 | 2.0V ~ 3.6V（标准 3.3V） |
| 封装 | LQFP-48（48 引脚） |

> 注意：STM32 使用 3.3V 供电，而 51 单片机使用 5V 供电。5V 电压不能直接给 STM32 供电，需要通过稳压芯片降至 3.3V 后再供电。USB 输出的 5V 电压同理，需要降压处理。

## STM32 片上资源（外设）

片上资源（外设）英文为 **Peripheral**，通过程序配置外设来完成各种功能。

> 以下介绍仅为入门级印象，后续课程会逐步深入。
### 内核级外设

| 外设 | 全称 | 功能说明 |
|------|------|---------|
| **NVIC** | 嵌套向量中断控制器 | 内核中用于管理中断、配置中断优先级 |
| **SysTick** | 系统滴答定时器 | 内核定时候钟，为操作系统提供定时服务（任务切换），本课程可用来实现 `delay_ms()` 函数 |

### 内核级外设（RCC 及之后为芯片级外设）

| 外设 | 全称 | 功能说明 |
|------|------|---------|
| **RCC** | 复位和时钟控制 | 配置系统时钟，使能各外设时钟。**STM32 上电后外设默认无时钟，不给时钟操作外设无效（目的是降低功耗），操作外设前必须先使能时钟** |
| **GPIO** | 通用IO口 | 最基本功能：点灯、读取按键等 |
| **AFIO** | 复用IO口 | 完成复用功能端口的重定义、中断端口的配置 |
| **EXTI** | 外部中断 | 配置后，当引脚有电平变化时触发中断，让 CPU 处理任务 |
| **TIM** | 定时器 | STM32 最常用、功能最多的外设。分为高级定时器、通用定时器、基本定时器三种。可完成定时中断、测频率、生成 PWM 波形、配置编码器接口等（电机驱动、舵机驱动必备） |
| **ADC** | 模数转换器 | 内置 12 位 AD 转换器，直接读取 IO 口的模拟电压值，无需外部 ADC 芯片 |
| **DMA** | 直接内存访问 | 帮助 CPU 完成大量数据搬运的繁杂任务 |
| **USART** | 同步/异步串口 | 平时用的 UART 即异步串口，USART 同时支持异步和同步串口 |
| **I2C** | I²C 总线 | 常用通信协议，内置硬件控制器，可硬件输出时序波形，也可用 GPIO 模拟时序 |
| **SPI** | SPI 总线 | 常用通信协议，同上 |
| **CAN** | CAN 总线 | 常用于汽车领域通信 |
| **USB** | USB 总线 | 可做成模拟鼠标、模拟 U 盘等设备 |
| **RTC** | 实时时钟 | 内部完成年月日时分秒计时，可接外部备用电池，掉电也能正常运行 |
| **CRC** | CRC 校验 | 数据校验，用于判断数据正确性 |
| **PWR** | 电源控制 | 让芯片进入睡眠等模式省电 |
| **BKP** | 备份寄存器 | 掉电时由备用电池保持数据 |
| **IWDG** | 独立看门狗 | 单片机死机或程序死循环时自动复位，保证系统稳定 |
| **WWDG** | 窗口看门狗 | 同上（窗口模式） |
| **DAC** | 数模转换器 | IO 口直接输出模拟电压，ADC 的逆过程 |
| **SDIO** | SD 卡接口 | 读取 SD 卡 |
| **FSMC** | 可变静态存储控制器 | 扩展内存，或配置成其他总线协议 |
| **USB OTG** | USB 主机接口 | 让 STM32 作为 USB 主机，去读取其他 USB 设备 |

> 注意：以上是 STM32F1 **整个系列**的全部外设，**不是所有型号都拥有全部外设**。以 C8T6 为例：
> - 64KB Flash / 20KB SRAM
> - 3 个通用定时器 + 1 个高级定时器（无基本定时器）
> - 2 个 SPI / 2 个 I2C / 3 个 USART / 1 个 USB / 1 个 CAN
> - 37 个 GPIO / 2 个 12 位 ADC（各 10 通道）
> - 无 DAC / SDIO / FSMC / USB OTG
>
> 具体外设数量需查阅对应型号的数据手册。

## STM32 系统结构

![STM32 系统结构图](STM32系统结构图.png)
> 图：STM32F103 系统结构全景图

系统结构大致可分为四个部分：

### 1. Cortex-M3 内核
内核引出三条总线：
- **I-Code（指令总线）**：连接 Flash 闪存，加载程序指令
- **D-Code（数据总线）**：连接 Flash，加载常量和调试参数
- **System（系统总线）**：连接 RAM、FSMC 等其他外设

### 2. AHB 系统总线
挂载最基本或性能最高的外设，如复位时钟控制（RCC）、SDIO 等。

### 3. APB 总线桥接
由于 AHB 和 APB 在协议、速度、数据传送格式上有差异，中间需要两个桥接来完成数据转换和缓存：
- **APB2**：性能较高，与 AHB 同频率（72MHz），连接重要外设，如 GPIO、USART、SPI、I2C、TIM、ADC、EXTI、AFIO 等
- **APB1**：性能较低，一般 36MHz，连接次要外设，如 I2C2/3/4、SPI2/3、UART4/5、DAC、PWR、BKP 等

### 4. DMA
可视为内核 CPU 的"小秘书"。大量数据搬运任务（如 ADC 连续转换的数据转运）若由 CPU 完成会浪费大量时间。DMA 通过 DMA 总线连接到总线矩阵，拥有和 CPU 一样的总线控制权，外设通过请求线发送 DMA 请求后，DMA 获得总线控制权并完成数据转运，**整个过程不需要 CPU 参与**，CPU 可专心处理其他任务。

## STM32F103C8T6 引脚定义

> 重要参考资料，可保存图片备用。
![STM32F103C8T6 引脚定义表|503](STM32F103C8T6引脚定义表.png)
> 图：STM32F103C8T6 引脚定义详细表

### 引脚类型说明

| 类型 | 含义 |
|------|------|
| S | 电源（VSS = 负极，GND；VDD = 正极，接 3.3V） |
| I | 输入 |
| O | 输出 |
| I/O | 输入输出 |

- **FT**：能容忍 5V 电压；**无 FT** 只能容忍 3.3V（接 5V 需加电平转换电路）
- **主功能**：上电后默认功能（一般与引脚名称相同，若不同以主功能为准）
- **默认复用功能**：IO 口同时连接的外设功能引脚
- **重定义功能**：当两个复用功能复用在同一 IO 口时，可将其中一个映射到其他端口

### 关键引脚速查

| 引脚号 | 引脚名称 | 主要功能 |
|--------|----------|---------|
| 1 | VBAT | 备用电池供电（接 3V 纽扣电池），断电时给 RTC 和备份寄存器供电 |
| 2 | PC13-TAMPER-RTC | GPIO / 侵入检测（防拆）/ RTC 输出 |
| 3-4 | PC14-OSC32_IN / PC15-OSC32_OUT | GPIO / 32.768kHz RTC 晶振 |
| 5-6 | OSC_IN / OSC_OUT | 主晶振（8MHz），经 PLL 倍频到 72MHz |
| 7 | NRST | 系统复位引脚（低电平复位） |
| 8-9 | VSSA / VDD | 模拟部分电源负极 / 正极（3.3V） |
| 10-19 | PA0-PA10 等 | GPIO / 复用功能，其中 PA0 兼 WKUP（待机唤醒）功能 |
| 20 | BOOT0 | **启动模式配置引脚** |
| 21-22 | VSS_1 / VDD_1 | 系统主电源（分区供电） |
| 23-33 | PA / PB / PC 端口 GPIO | 通用 IO 口 |
| 27-40 | PA13/14, PB3/4, PA15 等 | **调试端口**：SWD（PA13=SWDIO, PA14=SWCLK，需 2 根线）或 JTAG（5 根线） |
| 34-40 | 多个 GPIO | 复用调试端口或普通 IO 口 |
| 44 | BOOT1 | **启动模式配置引脚**（与 PB2 复用，上电瞬间为 BOOT1 功能） |
| 45-46 | 多个 GPIO | 通用 IO 口 |

> 使用 SWD 调试时，PA15、PB3、PB4 默认不作为普通 IO 口使用，需在程序中配置切换。

## STM32 启动配置

通过 **BOOT0** 和 **BOOT1** 两个引脚配置三种启动模式：

| BOOT0 | BOOT1 | 启动模式 | 说明 |
|--------|--------|---------|------|
| 0 | 任意 | 主闪存存储器模式 | **最常用**，正常执行 Flash 程序 |
| 1 | 0 | 系统存储器模式 | 内置 BootLoader，通过串口下载程序（用于 SWD 调试端口全部被配置为普通 IO 口后的救急下载） |
| 1 | 1 | 内置 SRAM 模式 | 用于程序调试，现阶段使用较少 |

### 何时使用串口下载？
- SWD 调试端口（PA13/14, PA15, PB3/4）全部被配置为普通 IO 口后，无法再用 ST-Link 下载程序
- 没有 ST-Link，只有串口模块时
- 操作：BOOT1=0, BOOT0=1，进入系统存储器模式，使用串口下载

> 注意：BOOT 引脚值仅在上电复位后的一瞬间有效，之后由程序锁定。PB2 在上电瞬间为 BOOT1 功能，第 4 个时钟周期后恢复为 PB2 功能。

## STM32 最小系统电路

单片机芯片无法独立工作，需要连接最基本的电路——**最小系统**。
![STM32 最小系统电路图|650](STM32最小系统电路.png)
> 图：STM32 最小系统完整电路

### 1. 电源电路

STM32 采用分区供电方式，主电源和模拟部分电源分布在芯片四周，需将所有 VSS（负极）接地，所有 VDD（正极）接 3.3V。每对 VDD 与 VSS 之间建议并联滤波电容（100nF）以稳定供电。

| 区域 | 引脚 | 建议 |
|------|------|------|
| VDD / VSS（第1组） | 9, 10 / 8, 11 | 3.3V 供电 + 100nF 滤波电容 |
| VDD / VSS（第2组） | 24, 30, 36 / 23, 29, 35 | 3.3V 供电 + 100nF 滤波电容 |
| VDDA / VSSA（模拟部分） | 9 / 8 | 3.3V 供电 + 100nF 滤波电容 |
| VBAT | 1 | 接 3V 纽扣电池（RTC 和备份寄存器供电），不需要时接 3.3V 或悬空 |

5V 转 3.3V 常用稳压芯片：XC6204、XC6206、AMS1117 等。

### 2. 晶振电路

**主晶振（OSC_IN / OSC_OUT）**：
- 典型值：**8MHz**
- 通过内部 PLL（锁相环）倍频至 **72MHz** 作为系统主时钟
- 两端各接 20pF 起振电容，另一端接地

**RTC 晶振（OSC32_IN / OSC32_OUT）**：
- 典型值：**32.768kHz**
- 接 PC14 / PC15 引脚
- 选 32.768kHz 的原因：2^15 = 32768，内部 RTC 电路经过 2^15 分频即可得到 1 秒信号

> 图中主晶振旁并联了 1MΩ 电阻（非必需，影响不大）。

### 3. 复位电路

由 **10kΩ 电阻** 和 **0.1µF 电容** 组成上电复位电路：

- 上电瞬间电容无电 → 电容短路 → NRST 引脚为低电平 → 芯片复位
- 电容逐渐充满电（短路变断路）→ NRST 被上拉为高电平 → 正常工作
- 复位时间极短，几乎感觉不到

左侧并联一个复位按键：
- 按下 → 电容放电 → NRST 接地 → 产生低电平复位信号
- 松手 → NRST 回到高电平 → 芯片从复位转为工作

> 设备上常见的小孔复位按钮就是这个原理。

### 4. 启动模式配置

通过跳线帽或拨码开关，将 BOOT0 和 BOOT1 引脚接高电平（3.3V）或低电平（GND）。推荐使用拨码开关比插跳线帽更方便。

### 5. 下载接口（SWD）

使用 ST-Link 下载程序时，只需引出：
- **SWDIO**（PA13）
- **SWCLK**（PA14）
- **3.3V**（供电）
- **GND**（接地）

共 4 根线即可完成程序下载和调试。
