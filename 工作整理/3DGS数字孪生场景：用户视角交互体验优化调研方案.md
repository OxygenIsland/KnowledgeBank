## 调研背景
- **背景：** 用户反馈基础视口交互（旋转、缩放、平移）“不好用”、“易失控”，影响编辑效率。
## 现状剖析
根据向博反馈，目前存在的问题有以下三点：
	1、在高斯椭球渲染的区域中进行缩放穿越时，变化不线性（不影响基本操作，可以选择性的优化）
	2、选点不好选中，要多次调节（目前这个问题已经解决）
	3、以用户选中的点为旋转中心和缩放时的中心（小建议，需要调研）
## 行业竞品调研
针对现场反馈的旋转中心的问题，与市面上常见的第三人称交互逻辑的产品进行了如下对比：

| 维度                 | **当前（DWdeploy）**      | Blender              | AutoCAD         | UE5                    | Cesium   |     | LCC2            |
| ------------------ | --------------------- | -------------------- | --------------- | ---------------------- | -------- | --- | --------------- |
| **轴心确定方式**         | ⚠️ 屏幕中心射线 Raycast 命中点 | ✅ 3D Cursor / 选中物体中心 | ✅ 鼠标悬停点 Raycast | ✅ Interest Point（焦点物体） | ✅ 鼠标下地面点 |     | ✅ 鼠标悬停点 Raycast |
| **鼠标位置影响轴心**       | ❌ 不影响，始终为屏幕中心         | ✅                    | ✅               | ✅                      | ✅        |     | ✅               |
| **失焦 fallback 策略** | ⚠️ 用上次距离沿 forward 推算  | ✅ 保持上次 3D Cursor 位置  | ✅ 保持上次兴趣点       | ✅ 用选中物体包围盒中心           | ✅ 用地面投影点 |     | ✅ 用上次命中点        |

Spark 2.0采用第一人称视角，以自身为中心进行旋转
## 解决方案
对于问题1：根据距离自适应缩放速度，当相机与mesh的距离越远，缩放的步长越大（方便快速飞行），越靠近mesh，缩放的步长越小（方便用户进行精细的移动），所以缩放操作会有非线性的变化。这种"离得越远步长越大，越近步长越小"的设计是**三维软件的行业共识**，可以通过调整缩放的步长逻辑来优化缩放操作变化不平滑的问题

对于问题3：市面上常见的第三人称交互逻辑的产品旋转缩放时的轴心大部分都与鼠标点击时的位置相关，而不是固定的以屏幕中心为轴心，修改后对交互体验有一定的提升
