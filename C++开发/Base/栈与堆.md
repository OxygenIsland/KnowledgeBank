这是 C++ 和 C# 思维方式差异最大的地方:**C++ 默认没有 GC,堆内存需要你手动释放**。
## 栈(Stack)

```cpp
void Foo() {
    int a = 10;          // 栈上分配,函数结束自动销毁,速度极快
    MyStruct s;            // 局部对象也在栈上,出了作用域自动调用析构函数
}  // 函数结束,a 和 s 自动被回收,无需手动管理
```

特点:由编译器自动管理生命周期(进入作用域分配、离开作用域释放),速度快,但**大小有限**(默认通常 1MB~8MB,递归过深或大数组放栈上会**栈溢出 Stack Overflow**)。

## 堆(Heap)—— 需要手动 `new` / `delete`

```cpp
int* p = new int(10);    // 在堆上分配一个 int,初始化为 10,p 存放它的地址
std::cout << *p;
delete p;                  // 必须手动释放!否则内存泄漏
p = nullptr;                // 好习惯:释放后置空,防止"野指针"被误用

int* arr = new int[10];   // 堆上分配数组
delete[] arr;               // 数组要用 delete[],不能用 delete,否则行为未定义

```

对比 C#:

```csharp

// C# 里你写
var obj = new MyClass();
// 用完不用管,GC 会在合适的时机自动回收

// C++ 里你必须

MyClass* obj = new MyClass();
// ... 使用
delete obj;   // 忘记写 = 内存泄漏(这块内存永远不会被回收,直到程序退出)

```

## 6.3 栈 vs 堆 对比表

|      | 栈 Stack           | 堆 Heap                       |
| ---- | ----------------- | ---------------------------- |
| 分配方式 | 直接声明变量,如 `int a;` | `new` 关键字                    |
| 释放方式 | 自动(出作用域)          | **手动 `delete` / `delete[]`** |
| 速度   | 快                 | 较慢(涉及内存分配器)                  |
| 大小限制 | 较小,容易溢出           | 较大,受系统内存限制                   |
| 生命周期 | 跟随作用域             | 跟随你手动控制,可以跨函数存活              |
| 典型用途 | 局部变量、小对象          | 大对象、生命周期不确定/需要跨函数传递的对象       |

## 6.4 常见内存问题(C# 程序员要重点建立警惕)

```cpp

// 1. 内存泄漏:忘记 delete
void Leak() {
    int* p = new int(1);
    // 忘了 delete p; → 这块内存再也回不来了
}

// 2. 悬空指针(dangling pointer):释放后继续使用

int* p = new int(1);
delete p;
std::cout << *p;   // 危险!p 已经被释放,这是"使用已释放内存",行为未定义
  

// 3. 野指针:指针没初始化就用

int* p;           // 没有初始化,p 指向一个随机地址
*p = 1;             // 危险!写入未知内存,可能崩溃可能悄悄破坏别的数据

// 4. 重复释放

int* p = new int(1);
delete p;
delete p;          // 危险!双重释放,行为未定义,可能崩溃
```

> **这正是为什么现代 C++(C++11 之后)极力推荐用"智能指针"代替裸指针 + 手动 new/delete** —— 见第 10 节。日常业务代码里,裸指针 + 手动管理已经是相对"过时"的写法了,但理解它的原理对你看懂底层代码、引擎源码(Unity 的 IL2CPP 产物、UE 源码)非常重要。
