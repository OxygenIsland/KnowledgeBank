## 什么是指针
变量存在内存里都有一个**地址**。指针就是一个**专门存地址的变量**。
```cpp
int a = 10;
int* p = &a;     // & 取地址符:取出 a 的内存地址,赋给指针 p
                  // p 的类型是 "int 的指针"

std::cout << a;   // 10
std::cout << p;   // 一串内存地址,如 0x7ffee...
std::cout << *p;  // *解引用:通过地址访问该地址存的值 → 10

*p = 20;          // 通过指针修改 a 的值

std::cout << a;   // 20
```

图解:
```
变量 a:  [ 内存地址 0x1000 ] → 存的值是 20

指针 p:  [ 内存地址 0x2000 ] → 存的值是 0x1000 (也就是 a 的地址)

  p ──────► a ──────► 20
   (存地址)   (存值)
```

## `nullptr`

```cpp
int* p = nullptr;   // 空指针,等价 C# 的 null
if (p == nullptr) {
    // 安全检查
}
*p = 1;             // 危险!对空指针解引用 = 程序崩溃(段错误),C# 里这是 NullReferenceException
```

C# 里所有引用类型默认就是"指针"(只是被语言隐藏了,空引用会抛异常被 try/catch 兜住),而 C++ 解引用空指针/野指针是**未定义行为**,轻则崩溃,重则悄悄破坏内存却不报错——这是 C++ 比 C# "危险"的核心原因。

## 引用(Reference)—— C++ 特有,C# 没有直接对应

```cpp
int a = 10;
int& ref = a;   // ref 是 a 的"别名",不是新变量,二者共用同一块内存

ref = 20;        // 等价于 a = 20

std::cout << a;  // 20

```

引用 vs 指针 关键区别:

|            | 指针 `int*`          | 引用 `int&`             |
| ---------- | ------------------ | --------------------- |
| 能否为空       | 可以(`nullptr`)      | **不可以**,定义时必须立即绑定一个变量 |
| 能否重新指向别的变量 | 可以                 | **不可以**,一旦绑定终生不变      |
| 语法         | 需要 `*` 解引用,`&` 取地址 | 像普通变量一样直接用,无需额外符号     |
| 本质         | 一个独立的、存地址的变量       | 编译器层面的"别名",通常不占额外内存   |
  
经验法则:**能用引用就用引用**(更安全、语法更简洁),**需要"可能为空"或"需要重新指向"或"需要做指针运算/数组遍历"时才用指针**。

## 指针运算

```cpp
int arr[3] = {10, 20, 30};
int* p = arr;        // 数组名会自动"退化"为指向首元素的指针

std::cout << *p;       // 10
p++;                   // 指针向后移动一个 int 的大小(不是移动1字节,而是按类型大小移动)
std::cout << *p;       // 20
std::cout << *(p + 1); // 30,等价于 arr[2]
```

这是数组下标 `arr[i]` 的底层原理:`arr[i]` 本质就是 `*(arr + i)`。

## 指针的指针 / 函数指针(了解即可)

```cpp
int a = 10;
int* p = &a;
int** pp = &p;     // 指向指针的指针,常用于需要修改"指针本身"的函数参数

// 函数指针:存放函数地址的变量
int Add(int a, int b) { return a + b; }

int (*funcPtr)(int, int) = Add;

std::cout << funcPtr(1, 2);  // 3,类似 C# 的 delegate,但更底层

```

## 智能指针(现代 C++ 的"半自动 GC")

裸指针 `new`/`delete` 容易内存泄漏,现代 C++(C++11 起)强烈推荐用**智能指针**自动管理堆内存生命周期,头文件 `<memory>`。
###  `std::unique_ptr` —— 独占所有权(最常用,默认首选)

```cpp
#include <memory>
std::unique_ptr<Player> p = std::make_unique<Player>(100, 50);  // 创建并管理一个 Player
p->TakeDamage(10);     // 像普通指针一样用 ->
// 离开作用域时,自动调用 delete,无需手动管理!
// unique_ptr 不能被拷贝(独占语义),只能"转移"
std::unique_ptr<Player> p2 = std::move(p);  // 所有权转移给 p2,p 变为空
```
心智模型:类似 C# 里"只有一个变量持有这个对象的引用",但**保证用完自动释放**,接近 `using` 语句的自动 `Dispose`,但更通用。
### `std::shared_ptr` —— 共享所有权(引用计数)

```cpp
std::shared_ptr<Player> p1 = std::make_shared<Player>(100, 50);
std::shared_ptr<Player> p2 = p1;   // 可以拷贝!引用计数变为 2
// 当最后一个 shared_ptr 被销毁(计数归零)时,对象才被真正 delete
```

心智模型:**最接近 C# 对象引用的行为**——多个变量可以指向同一个对象,内部用"引用计数"代替 GC,计数归零自动释放。区别是 C# 的 GC 能处理循环引用,`shared_ptr` 的引用计数**处理不了循环引用**(会导致永远不归零、内存泄漏),需要配合下面的 `weak_ptr` 解决。

###  `std::weak_ptr` —— 弱引用,解决循环引用

```cpp
class Node {
public:
    std::shared_ptr<Node> next;
    std::weak_ptr<Node> prev;     // 用 weak_ptr 打破循环引用,不增加引用计数
};

std::weak_ptr<Node> wp = someSharedPtr;

if (auto sp = wp.lock()) {        // 使用前需要 lock() 转成 shared_ptr,并检查对象是否还存在
    sp->DoSomething();
}
```
等价 C# 的 `WeakReference`。
### 选择建议

| 场景                     | 用什么                             |
| ---------------------- | ------------------------------- |
| 一般情况,对象只有一个"主人"        | `unique_ptr`(默认首选,零额外开销)        |
| 多处需要共享同一个对象            | `shared_ptr`                    |
| 需要引用但不想影响生命周期(如父子双向引用) | `weak_ptr`                      |
| 极致性能/嵌入式场景需要完全手动控制     | 裸指针 + 手动 `new`/`delete`(需要非常谨慎) |
  
> **重要提醒**:Unity/UE 的对象生命周期模型和裸 C++ 智能指针不完全一样——Unreal 有自己的一套垃圾回收(`UObject` 体系用 `UPROPERTY` 标记 + UE 自带 GC),不是用标准库智能指针;如果你学 C++ 是为了 UE,智能指针知识仍然必学(用于非 UObject 的普通 C++ 类),但 UObject 体系另有一套规则,见第 21 节。
