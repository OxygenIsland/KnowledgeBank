## 方式1
在sysb工程中进行编译，CICD一下，然后到gitlab中，LRSH-SDS-bsp/projects/robot-sdksdk-server-deployment/Pipelines中找到对应的release的镜像版本号，比如im.daystar.cloud:10443/daystar_bot/sdk_server:sysb_alpha_v2.5.2_release_humble_arm64_20260626
![[Pasted image 20260629113738.png|475]]

然后在机器人上`docker image pull`把镜像拉下来，docker images 找到对应镜像后，使用`docker-compose ls`查看对应的配置文件，如下所示：
![[Pasted image 20260629113408.png]]
修改对应配置文件：`sudo vim /opt/sysb/project/autoptz_v1.0.9/docker-compose.yml`
在配置文件中，修改sdk 的版本号：
![[Pasted image 20260629113659.png]]

## 方式2
进入到SDK的容器中
`docker exec -it sdk_server bash`
查看目录，可以看到只有一个install文件夹
![[Pasted image 20260629113951.png]]
新建一个src目录
![[Pasted image 20260629114113.png]]
将你修改过的模块的源码拷贝到这个目录下，然后进行编译
`overlay_build_release`
